# Snap Testing Guide

## 测试环境要求

- macOS 13.0 (Ventura) 或更高版本
- Xcode 15.0+ 或 Swift 5.9+
- 屏幕录制权限

## 功能测试清单

### 1. 权限测试

#### 1.1 首次运行权限请求
- [ ] 启动应用
- [ ] 验证屏幕录制权限请求弹出
- [ ] 打开系统设置 → 隐私与安全性 → 屏幕录制
- [ ] 验证 SnapApp 出现在列表中
- [ ] 授予权限

#### 1.2 权限被拒绝处理
- [ ] 拒绝屏幕录制权限
- [ ] 尝试截图
- [ ] 验证错误处理（应显示友好提示）

### 2. 截图功能测试

#### 2.1 区域截图
- [ ] 按下 `⌘ + ⇧ + A` 或从菜单选择"区域截图"
- [ ] 验证全屏半透明覆盖层显示
- [ ] 拖拽鼠标选择区域
- [ ] 验证选区高亮显示
- [ ] 释放鼠标
- [ ] 验证编辑器窗口打开
- [ ] 验证截图内容正确

**测试用例**:
- 小区域 (50x50 像素)
- 中等区域 (500x500 像素)
- 大区域 (1920x1080 像素)
- 从左上到右下拖拽
- 从右下到左上拖拽

#### 2.2 全屏截图
- [ ] 按下 `⌘ + ⇧ + S` 或从菜单选择"全屏截图"
- [ ] 验证编辑器窗口立即打开
- [ ] 验证截取整个屏幕内容
- [ ] 验证分辨率正确（Retina 屏幕应为 2x）

#### 2.3 取消截图
- [ ] 按下 `⌘ + ⇧ + A`
- [ ] 按 ESC 键
- [ ] 验证覆盖层关闭
- [ ] 验证编辑器未打开

### 9. 性能测试

#### 9.1 内存占用
1. 使用 Activity Monitor:
   - [ ] 启动应用，记录初始内存 (应 < 50MB)
   - [ ] 执行 10 次区域截图
   - [ ] 记录内存增长 (应 < 100MB)
   - [ ] 关闭所有编辑器
   - [ ] 验证内存恢复

2. 使用 Instruments:
   - [ ] 打开 Xcode → Product → Profile
   - [ ] 选择 Allocations 模板
   - [ ] 记录应用运行时的内存分配
   - [ ] 检查内存泄漏
   - [ ] 验证无持续增长的内存占用

#### 9.2 响应时间
- [ ] 触发截图，测量从按键到显示的时间
   - 目标: < 100ms
- [ ] 绘制标注，测量渲染时间
   - 目标: 60 FPS (16ms per frame)
- [ ] 导出图像，测量处理时间
   - 目标: < 200ms

#### 9.3 CPU 使用率
- [ ] 应用空闲时 CPU 使用率 (应 < 1%)
- [ ] 截图时 CPU 峰值 (应 < 50%)
- [ ] 编辑时 CPU 使用率 (应 < 30%)

## 自动化测试

### Unit Tests
```swift
// 示例测试用例
func testAnnotationCreation() {
    let annotation = Annotation(
        type: .rectangle,
        points: [CGPoint(x: 0, y: 0), CGPoint(x: 100, y: 100)],
        color: .red,
        lineWidth: 3
    )
    XCTAssertEqual(annotation.type, .rectangle)
    XCTAssertEqual(annotation.points.count, 2)
}

func testUndoRedo() {
    let appState = AppState()
    let annotation = Annotation(type: .line, points: [], color: .blue, lineWidth: 2)
    
    appState.addAnnotation(annotation)
    XCTAssertEqual(appState.annotations.count, 1)
    
    appState.undo()
    XCTAssertEqual(appState.annotations.count, 0)
    
    appState.redo()
    XCTAssertEqual(appState.annotations.count, 1)
}
```

## 性能基准

| 指标 | 目标值 | 测试方法 |
|------|--------|----------|
| 启动时间 | < 1s | 从启动到菜单栏图标显示 |
| 截图响应 | < 100ms | 从按键到覆盖层显示 |
| 内存占用（空闲） | < 50MB | Activity Monitor |
| 内存占用（编辑） | < 150MB | 编辑 1920x1080 截图 |
| CPU 占用（空闲） | < 1% | Activity Monitor |
| 导出时间 | < 200ms | 含 10 个标注的截图 |
